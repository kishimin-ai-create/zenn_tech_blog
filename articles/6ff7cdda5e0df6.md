## 作ったもの

日記アプリを GitHub Copilotと Codex に作らせました。

**アプリ概要**
作ったのは、日記を公開閲覧できるフルスタックの Web アプリです。

一般ユーザーは日記の一覧と詳細を閲覧できます。管理者はログイン後、管理画面から日記の作成、編集、削除ができます。UI は日本語と英語に対応しています。フロントエンドとバックエンドを分けた構成で実装しました。

バックエンドでは日記 API と認証 API を用意し、OpenAPI からフロントエンド用の API クライアントを生成しています。AI に実装を任せるだけでなく、実装結果のレビューやテストも含めて、AI を使った開発プロセス全体を試す題材として作りました。

## 工夫した点

AIのハーネスとして、 skills を作成しました。
AIが適切な指示を実行できるように、 instructions と skills の分離を行いました。
セキュリティを考慮した実装を行えるように、 OWASP ASVS を用いたセキュリティに関する skills を追加しました。
AIに実装させたいものを明確にするために、仕様書と設計書を ChatGPT と作成しました。
一般的なWebアプリらしいUIにするために、一般的なWebアプリのデザインにするよう指示し、 DESIGN.md を用いて自由にデザインさせました。

## 学び

今回の開発では、意図したコーディングにならない場面がありました。
リントや型チェックを通し、仕様を満たすことを優先する傾向があると感じました。
指示が不足していると、人であれば避けるようなハードコーディングを行うことがありました。
可読性についても、明示的に指示しないと十分に考慮されないと感じました。
